## Overview

NLP skill files generate projects that showcase practical text processing and language model integration: from classical NLP pipelines through transformer-based classification and generation to retrieval-augmented production services. NLP projects are high-signal because they demonstrate both ML fluency and the ability to handle the messiness of real text data — tokenization edge cases, entity disambiguation, multilingual inputs, and latency/quality tradeoffs. Generated projects use pretrained models from HuggingFace Hub and include offline-capable fixtures so the sandbox can run tests without network access.

## Architecture Patterns

All NLP projects generated by this skill MUST follow these patterns:

1. **Text preprocessing is a versioned pipeline.** Tokenization, lowercasing, stop-word removal, and normalization steps are encapsulated in a `TextPreprocessor` class with a version string. Tests verify that the same input produces the same output across runs.

2. **Models and tokenizers are loaded once at startup.** Never load a model inside a request handler. Use application lifespan events (`@app.on_event("startup")`) or dependency injection to load once and reuse.

3. **Batch inference is supported.** Every inference endpoint accepts either a single string or a list of strings and processes them in a single batch. Single-item requests are wrapped in a list internally.

4. **Embeddings are cached.** For corpus-based systems (search, clustering), embeddings are persisted and only recomputed when the source document changes (content-hash based invalidation).

5. **Model cards are committed.** A `model_card.md` file documents: model name, source, license, training data, evaluation metrics, known limitations, and inference configuration.

6. **Evaluation metrics are reproducible.** An `eval/` directory contains evaluation scripts and fixed test sets. Running `python eval/run_eval.py` produces a `results.json` with precision, recall, F1, and latency metrics.

## Quality Criteria

The QualityGate checks the following for NLP projects:

1. **Tests pass.** `pytest tests/ -v` exits with code 0.
2. **Linting passes.** `ruff check .` reports zero errors.
3. **Type checking passes.** `mypy --strict src/` exits with code 0.
4. **Model card exists and is non-empty.** `model_card.md` must contain at minimum: model name, source URL, license, and evaluation metrics.
5. **Batch inference is tested.** At least one test verifies that processing a batch of N items produces the same results as processing N individual items.
6. **Evaluation script runs.** `python eval/run_eval.py --sample` completes without error using the bundled sample test set.
7. **No model weights committed.** `.gitignore` must exclude `*.bin`, `*.safetensors`, `*.pt` files.

## ADR Templates

### ADR-001: Model Selection and Hosting

**Status:** Accepted

**Context:** The project requires a pretrained language model for [classification/embedding/generation]. We must balance model quality, inference latency, and operational simplicity for a portfolio demonstration.

**Decision:** Use HuggingFace Transformers with models downloaded from HuggingFace Hub at startup. For classification tasks: `distilbert-base-uncased` (66M params, fast CPU inference). For embeddings: `sentence-transformers/all-MiniLM-L6-v2` (22M params, 384-dim, excellent quality/speed tradeoff). Local caching via `TRANSFORMERS_CACHE` environment variable.

**Consequences:** We get state-of-the-art model quality without hosting infrastructure. Cold-start latency includes one-time model download (~250MB for DistilBERT). Subsequent starts use the local cache.

### ADR-002: Text Chunking Strategy

**Status:** Accepted

**Context:** Long documents must be split into chunks before embedding or classification. Naive fixed-size chunking can split mid-sentence, degrading quality. Semantic chunking is expensive. We need a pragmatic default.

**Decision:** Sentence-aware chunking with a target of 512 tokens and 64-token overlap. Use `spaCy` sentence boundary detection to avoid splitting mid-sentence. The chunk size matches the max sequence length of most BERT-family models.

**Consequences:** We preserve sentence coherence at chunk boundaries and get natural overlap for retrieval. Chunking adds ~5ms per document. Documents with very long sentences (>512 tokens) are split at the token boundary as a fallback.

### ADR-003: Embedding Cache Strategy

**Status:** Accepted

**Context:** Re-embedding a large corpus on every service restart is expensive (both latency and compute cost). We need a durable cache with cache invalidation.

**Decision:** Store embeddings in a FAISS flat index persisted to disk, keyed by a SHA-256 content hash of the source text. At startup, load the persisted index. On new documents, embed and add to the index. When source text changes (hash mismatch), re-embed that document only.

**Consequences:** Service restarts are fast (index loads in seconds for <1M documents). Cache is automatically invalidated when content changes. The index file must be excluded from `.gitignore` since it is a derived artifact.

## Common Failure Modes

1. **HuggingFace model download fails in sandbox.** The model download requires outbound internet access. If the sandbox network policy blocks HTTPS, pre-download models to `/mnt/models/` and set `TRANSFORMERS_CACHE=/mnt/models`. Fix: always test with `--offline` flag in CI.

2. **Tokenizer max length overflow.** Inputs longer than the model's max sequence length (typically 512 tokens) raise an exception or silently truncate. Fix: add explicit truncation with `truncation=True, max_length=512` in all `tokenizer()` calls.

3. **Memory leak from repeated model loading.** Loading models inside loops or request handlers causes OOM. Fix: use a module-level singleton or FastAPI dependency injection with `lru_cache`.

4. **Non-deterministic test failures due to random sampling.** Generation with `do_sample=True` produces different outputs each run. Fix: set `seed=42` in all generation calls used in tests, or test only deterministic properties (e.g., output length constraints).

5. **FAISS index corruption on concurrent writes.** FAISS flat indices are not thread-safe for simultaneous writes. Fix: use a write lock (`asyncio.Lock`) around all index mutation operations.
